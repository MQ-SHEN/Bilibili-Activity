# Bilibili-Activity-Data-Collection-Assistant
主要功能：爬取B站活动数据
一、 技术架构与核心库
本项目采用 Python 语言开发，结合了前端 GUI 交互、浏览器自动化控制和数据持久化处理。

1. 核心依赖库
Selenium (WebDriver):
功能：核心爬虫引擎。通过模拟真实用户的浏览器行为（点击、滚动、等待），绕过复杂的动态渲染页面，抓取 B 站活动的实时数据。

Tkinter & ttk:
功能：构建跨平台的桌面图形界面（GUI）。提供窗口、按钮、进度条、日志框等交互控件。

tkcalendar:
功能：提供高定制化的日历选择控件，用于过滤活动开始日期。

openpyxl:
功能：Excel 文件处理库。负责将抓取到的列表数据写入 .xlsx 文件，并设置表头样式、列宽及超链接。

webdriver-manager:
功能：自动管理驱动。程序启动时会自动检测本地浏览器版本，并从云端下载匹配的驱动（ChromeDriver/EdgeDriver），解决了手动配置驱动报错的痛点。

二、 核心功能模块设计
1. 智能日期筛选过滤系统
技术点：多格式日期解析。
详情：程序内置了 parse_date_str 函数，能够自动识别 B 站页面上多种不规范的日期格式（如：2024-05-20、2024年5月20日、05/20 等）。用户在 GUI 选定日期后，爬虫会自动忽略所有早于该日期的活动，确保数据的有效性。

2. 自动化驱动自适配引擎
技术点：webdriver_manager 集成。
详情：传统的 Selenium 工具经常因为浏览器更新导致驱动失效。本项目实现了“自动匹配”与“手动指定”双模式。自动模式下，程序会静默完成驱动的下载与配置，极大降低了普通用户的使用门槛。

3. 异步多线程 UI 保护
技术点：threading 模块。
详情：为了防止爬虫在运行过程中导致 GUI 窗口卡死（假死），本项目将爬虫逻辑封装在独立的 worker_thread 线程中，通过回调函数实时更新主线程的进度条和日志。

4. 规范化数据导出与富文本处理
技术点：Excel 样式定制。
详情：导出的 Excel 不仅仅是纯文本。程序会对表头进行背景上色（B站蓝）、自动调整列宽以适应长标题，并利用 openpyxl 的 hyperlink 功能为活动标题直接嵌入网页链接，实现“点击即打开”。

三、 针对 Python 3.13 的稳定性优化
在最新的环境适配中，本项目针对 Tkinter 机制做了特殊处理：

独立模态窗口方案：避开了 tkcalendar 在 Python 3.13 中常见的下拉框焦点冲突闪退问题，改用 Toplevel 级联窗口并调用 grab_set() 锁定交互。

资源路径自动映射：通过 resource_path 函数，确保程序在打包成独立 EXE 后，内部图片资源（ICO 图标）路径不会因解压位置改变而失效。

四、 功能清单总结

功能模块,实现描述
多浏览器支持,同时支持 Chrome、Edge、Firefox 三大主流浏览器切换。
增量采集,通过日期对比功能，仅采集近期发布的新活动。
实时日志,毫秒级显示当前爬取状态，错误自动捕获并弹窗提醒。
配置记忆,自动保存上次的保存路径、浏览器偏好，下次打开即用。
断点保护,提供“停止”按钮，手动中止任务时确保已采集的数据不会丢失。
